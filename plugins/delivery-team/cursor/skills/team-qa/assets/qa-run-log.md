# QA Run Log (cross-project fallback, append-only)

> `qa-strategist` appends here **only** when the project doesn't name its own
> QA run-log in `PROJECT-CONTEXT.md`. A project-specific log (living in that
> project's own repo) is always preferred — it keeps a project's QA history
> with the project, not mixed in with every other project's. This file exists
> so QA runs against a project with no such convention configured still get a
> record somewhere, not lost.

> The *catalog* of recurring failure modes, if a project has one, is that
> project's own — shared with team-intake, read and updated by both. There is
> no cross-project catalog here (a defect catalog is inherently
> project-specific — nothing to fall back to).
