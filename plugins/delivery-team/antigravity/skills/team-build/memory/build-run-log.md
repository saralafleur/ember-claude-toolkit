# Build Run Log (cross-project fallback, append-only)

> `build-lead` appends here **only** when the project being built doesn't name
> its own build run-log in `PROJECT-CONTEXT.md`. A project-specific log (living
> in that project's own repo) is always preferred — it keeps a project's build
> history with the project, not mixed in with every other project's. This file
> exists so builds against projects with no such convention configured still
> get a record somewhere, not lost.
