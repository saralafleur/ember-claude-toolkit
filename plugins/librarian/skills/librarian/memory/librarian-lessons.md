# Librarian lessons — read this FIRST on every `/librarian audit`

> How to be a better librarian. The analyst adds a dated note whenever an audit
> surfaces a recurring capture miss, a description anti-pattern, or a taxonomy
> lesson. Keep entries terse. A lesson earns a place when it has bitten us twice
> — or once, if it's structurally likely to repeat.

---

## Description anti-patterns (these cause retrieval misses)
- **Topic-naming instead of trigger-naming.** "About the archive utility" fails;
  "Load when archiving/restoring TOPR data or exporting a snapshot — covers the
  archive utility CLI and its flags" succeeds. Descriptions must echo the *words a
  future task will use*, not just the subject.
- **Too broad → always loads → noise.** If an entry matches almost everything, it
  matches nothing useful. Narrow the trigger.
