# librarian — self-memory

This is the librarian's memory about **how to be a better librarian**. It is the
analog of the `team-*` skills' `recurring-issues.md`. The `librarian-analyst`
reads it FIRST on every `/librarian audit` and updates it (Sara-gated).

- **`librarian-lessons.md`** — recurring capture misses, description anti-patterns,
  taxonomy lessons, and process improvements that have proven out. Terse and
  high-signal: this is read on every audit, so bloat defeats it.

Companion logs (in `../audit/`):
- **`audit-log.md`** / **`description-history.md`** — changes to library *content*.
- **`self-audit-log.md`** — changes to the librarian's *own* files (SKILL.md,
  agents, templates, taxonomy), with thrashing detection on its own evolution.
